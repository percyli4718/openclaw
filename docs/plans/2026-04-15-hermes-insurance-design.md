**版本**：v0.1.0  
**日期**：2026 年 4 月 15 日  
**状态**：待审查

---

## 一、产品概述

### 1.1 产品定位

基于 Hermes Agent 打造的 AI+ 保险获客系统，采用"智能体 + 培训 + 课程 + 企业深度服务"四位一体模式，帮助保险从业者解决获客痛点。

### 1.2 目标用户

| 用户类型 | 规模 | 核心需求 |
|---------|------|---------|
| 保险代理人/经纪人 | 个人从业者 | 快速获得潜在客户线索 |
| 团队长 | 管理 10-100 人团队 | 规模化获客、团队赋能 |
| 保险公司/经纪公司 | 企业客户 | 系统化解决方案、数据沉淀 |

### 1.3 核心价值

用 AI 解决保险行业最大的刚需痛点——**获客**。

- 传统方式：陌拜、转介绍、朋友圈刷屏，效率低、成本高
- AI 方案：智能体自动生成获客内容、7×24 小时自动化跟进、数据驱动策略优化

### 2.2 技术栈选型

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **前端框架** | uni-app + Vue 3 + TypeScript | 一套代码编译多端 |
| **UI 组件库** | uView UI 2.0 + 自定义组件 | 全平台兼容 |
| **状态管理** | Pinia | Vue 3 官方推荐 |
| **后端框架** | FastAPI + Hermes Agent | Python 3.11+ |
| **AI 模型** | 混合部署：Claude API + Ollama 本地 | SaaS 用云端，私有化用本地 |
| **数据库** | PostgreSQL 15 | 主业务数据库 |
| **缓存** | Redis 7 | 会话缓存、限流计数 |
| **向量库** | Qdrant 1.7 | 客户画像向量检索 |
| **部署** | Docker Compose | SaaS 集中 + 私有化轻量 |

## 三、核心功能拆解

### 3.1 MVP 功能范围（v0.1.0）

**包含平台**：Web + 微信小程序

#### 功能 1：智能体获客内容生成

| 子功能 | 说明 | 优先级 |
|-------|------|--------|
| 朋友圈文案生成 | 根据保险产品/活动，自动生成多条朋友圈文案 | P0 |
| 短视频脚本生成 | 生成 15-60 秒保险科普短视频脚本 | P0 |
| 海报文案生成 | 营销海报标题 + 正文 + CTA | P1 |
| 话术模板库 | 预置 100+ 保险销售话术模板 | P1 |

**Hermes Agent 技能**：
- `generate_wechat_copywriting`：朋友圈文案生成
- `generate_short_video_script`：短视频脚本生成
- `generate_poster_copywriting`：海报文案生成
- `get_sales_script_template`：获取话术模板

#### 功能 2：AI 客户画像分析

| 子功能 | 说明 | 优先级 |
|-------|------|--------|
| 客户标签管理 | 手动/自动打标签（年龄、职业、家庭状况等） | P0 |
| 客户分层 | 基于消费能力、保险意识自动分层 | P0 |
| 需求预测 | AI 分析客户潜在保险需求 | P1 |
| 客户检索 | 向量检索相似客户、高价值客户 | P1 |

**Hermes Agent 技能**：
- `analyze_customer_profile`：客户画像分析
- `segment_customers`：客户分层
- `predict_insurance_needs`：需求预测
- `search_similar_customers`：相似客户检索

#### 功能 3：自动化跟进

| 子功能 | 说明 | 优先级 |
|-------|------|--------|
| 跟进计划制定 | AI 根据客户分层制定跟进节奏 | P0 |
| 定时消息推送 | 7×24 小时自动发送关怀/产品消息 | P0 |
| 跟进记录 | 自动记录每次跟进内容、客户反馈 | P0 |
| 跟进提醒 | 到期未跟进客户提醒人工介入 | P1 |

**Hermes Agent 技能**：
- `create_followup_plan`：制定跟进计划
- `schedule_automated_message`：定时消息推送
- `log_followup_record`：跟进记录
- `notify_overdue_followup`：逾期提醒

### 3.2 V2 功能规划（v0.2.0）

#### 功能 4：数据驱动的获客策略优化

| 子功能 | 说明 |
|-------|------|
| 获客效果分析 | 统计各渠道、各内容类型的转化率 |
| A/B 测试 | 自动测试不同话术、海报的效果 |
| 策略推荐 | AI 推荐最优获客策略 |
| 竞争对手分析 | 监控同业获客策略（需合规） |

#### 平台扩展

- iOS/Android/鸿蒙 App（uni-app x 编译）
- 平板适配（横屏布局）

## 四、核心流程设计

### 4.1 用户获客工作流

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户操作                                  │
│                                                                 │
│  1. 输入客户基本信息 ──→ 2. AI 生成获客内容 ──→ 3. 一键发送     │
│     (小程序/Web)         (Hermes Agent)        (微信/朋友圈)     │
│                              │                                  │
│                              ▼                                  │
│  6. 查看转化报告 ←── 5. 跟进记录 ←── 4. 客户互动追踪           │
│     (数据分析)          (自动记录)          (AI 监控)            │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 Hermes Agent 技能调用流程

```
用户请求 ──→ API Gateway ──→ Router ──→ Skill Executor
                                              │
                    ┌─────────────────────────┼──────────────────┐
                    │                         │                  │
                    ▼                         ▼                  ▼
            Content Generator         Customer Analyst     Followup Manager
            (内容生成技能)           (客户分析技能)        (跟进管理技能)
                    │                         │                  │
                    ▼                         ▼                  ▼
              Claude API / Ollama      PostgreSQL           Redis Scheduler
              (AI 模型推理)             (客户数据存储)        (定时任务)
```

### 4.3 小程序 AI 调用合规流程

```
┌──────────────┐     ┌──────────────┐     ┌──────────────────┐
│   微信小程序  │     │   API Gateway │     │  Hermes Agent    │
│              │     │               │     │                  │
│  用户操作 ──┼──→ │  请求验证 ──┼──→ │  技能执行        │
│              │     │  (鉴权/限流)  │     │                  │
│              │←──┼── 响应数据 ──┼──← │  AI 模型调用        │
│  渲染结果    │     │  (JSON)       │     │  (后端转发)       │
└──────────────┘     └──────────────┘     └──────────────────┘

注意：小程序端不直接调用 AI 模型 API，所有 AI 推理在后端完成
```

## 五、风险识别

### 5.1 技术风险

| 风险 | 概率 | 影响 | 缓解方案 |
|------|------|------|---------|
| uni-app x 成熟度不足 | 中 | 中 | MVP 用传统 uni-app，V2 再评估 x 版本 |
| Hermes Agent 技能迁移性问题 | 中 | 中 | 技能依赖明确模型版本，避免跨模型使用 |
| 本地模型效果不如云端 | 高 | 中 | 私有化客户提供云端/本地切换选项 |
| 小程序审核不通过 | 中 | 高 | 提前预审，准备 H5 兜底方案 |

### 5.2 合规风险

| 风险 | 概率 | 影响 | 缓解方案 |
|------|------|------|---------|
| 保险销售话术合规 | 高 | 高 | 预置话术经法律顾问审核，支持自定义审核 |
| 客户数据隐私保护 | 高 | 高 | 数据加密存储、权限分级、审计日志 |
| AI 生成内容责任 | 中 | 中 | 明确告知 AI 生成，人工确认后再发送 |

### 5.3 商业风险

| 风险 | 概率 | 影响 | 缓解方案 |
|------|------|------|---------|
| 大厂进场（12-18 个月窗口） | 高 | 高 | 快速占领细分赛道，建立品牌认知 |
| 获客成本超预期 | 中 | 中 | 聚焦行业社群、线下会议等低成本渠道 |
| 付费转化率低 | 中 | 中 | 免费试用 + 标杆案例 + 培训转化组合拳 |

## 六、边界条件

### 6.1 不做的事情

1. **不直接对接保险公司核心系统**（需资质、周期长）
2. **不做理赔辅助功能**（与获客无关，聚焦核心）
3. **不做 CRM 全功能**（专注获客场景，不做通用 CRM）
4. **不提供法律/医疗建议**（AI 生成内容仅供销售参考）

### 6.2 依赖条件

| 依赖项 | 说明 | 状态 |
|-------|------|------|
| Hermes Agent 稳定性 | 需验证单节点部署稳定性 | 待验证 |
| 微信小程序资质 | 需保险相关行业资质 | 待申请 |
| AI 模型 API | 需稳定的 Claude API/国产大模型 | 已确认 |
| uni-app 生态 | 需 uView UI 等组件库支持 | 已确认 |

### 6.3 成功标准

| 指标 | 目标值 | 时间 |
|------|--------|------|
| MVP 发布 | Web + 小程序可用 | 2 个月 |
| 付费客户 | 10 家付费客户 | 3 个月 |
| 客户续费率 | > 60% | 6 个月 |
| 标杆案例 | 3 个行业标杆案例 | 4 个月 |
