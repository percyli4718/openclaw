## 一、项目背景与定位

### Q1: 请介绍一下保客通项目的背景和定位？

**参考答案**:

保客通是基于 Hermes Agent 打造的 AI+ 保险获客系统，采用"智能体 + 培训 + 课程 + 企业深度服务"四位一体模式。

**业务背景**:
- 保险行业获客难：传统陌拜、转介绍效率低、成本高
- 代理人需要 AI 工具：自动生成获客内容、7×24 小时自动化跟进
- 市场上缺少垂直领域的 AI 获客解决方案

**保客通定位**:
- AI 保险顾问成长平台（不只是获客工具）
- 四种交付形态：Web 应用 + 微信小程序 + App+ 企业深度服务
- 三种客户类型：小微 (2999 元/年)、中型 (19999 元/年)、大型 (50000 元/年起)

**核心功能**:
1. 智能体获客内容生成（朋友圈文案、短视频脚本、海报文案）
2. AI 客户画像分析（标签管理、分层分析、需求预测）
3. 自动化跟进（跟进计划、定时消息、跟进记录）
4. 数据驱动策略优化（获客效果分析、A/B 测试）

**目标客户**:
- 保险代理人/经纪人（个人从业者）
- 团队长（管理 10-100 人团队）
- 保险公司/经纪公司（企业客户）

## 二、技术架构（对应 resume 技术栈）

### Q2: 保客通的技术架构是怎样的？

**参考答案**:

采用 Hermes Agent + FastAPI + uni-app 架构，支持 SaaS 多租户和私有化部署双模式。

**前端层**:
- uni-app + Vue 3 + TypeScript
- 一套代码编译多端：Web + 微信小程序 + iOS/Android/鸿蒙 App
- uView UI 2.0 + 自定义组件
- Pinia 状态管理

**后端层**:
- FastAPI + Hermes Agent
- Python 3.11+
- 技能引擎：内容生成、客户分析、跟进管理
- 记忆系统：短期会话上下文 + 长期向量存储

**数据存储层**:
- PostgreSQL 15: 业务数据（支持 RLS 行级安全）
- Redis 7: 会话缓存、定时任务调度（Cluster 高可用）
- Qdrant 1.7: 客户画像向量检索

**AI 模型层**:
- 混合部署：云端 Claude API (SaaS) + 本地 Ollama(私有化)
- Circuit Breaker 熔断器模式（防止 AI 调用级联故障）

**部署架构**:
- SaaS 模式：Docker Compose 多租户共享
- 私有化模式：独立 PostgreSQL 实例，数据物理隔离

## 三、核心功能实现

### Q3: Hermes Agent 技能是如何实现的？

**参考答案**:

实现了 7 个核心技能，每个技能都有明确的输入输出 Schema 和 SLA 定义。

**内容生成技能**:
```python
# generate_wechat_copywriting 输入 Schema
{
  "product_name": {"type": "string", "required": true},
  "product_type": {"type": "string", "enum": ["重疾险", "医疗险", "寿险", "意外险"]},
  "tone": {"type": "string", "enum": ["专业", "亲和", "幽默", "紧迫"], "default": "专业"},
  "count": {"type": "integer", "minimum": 1, "maximum": 5, "default": 3}
}

# 输出 Schema
{
  "status": {"type": "string", "enum": ["success", "error"]},
  "data": {
    "copies": [
      {"id": "string", "content": "string", "hashtags": "array", "score": "number"}
    ]
  },
  "duration_ms": {"type": "integer"}
}
```

**SLA 定义**:
- 朋友圈文案生成：预期 3-8 秒，超时 15 秒，降级到预置模板
- 短视频脚本生成：预期 5-12 秒，超时 20 秒
- 客户画像分析：预期 2-5 秒，超时 10 秒

**技能执行流程**:
1. 用户请求 → API Gateway(认证/限流)
2. Router 路由到对应技能
3. Hermes Agent 执行技能（调用 AI 模型或工具）
4. 结果返回并记录审计日志

## 四、安全与合规

### Q4: 如何保证数据安全和合规？

**参考答案**:

从 5 个层面保障安全合规：

**1. 数据隔离**:
- PostgreSQL RLS (Row-Level Security) 行级安全策略
- 租户数据通过 tenant_id 逻辑隔离
- 中型/大型客户独立 database，物理隔离

**2. 敏感数据加密**:
- 客户手机号/身份证/地址：AES-256 字段级加密
- 密钥管理：AWS KMS 或本地 Vault
- 用户密码：bcrypt 哈希加盐存储

**3. AI 调用安全**:
- Circuit Breaker 熔断器模式（防止级联故障）
- 超时降级策略（AI 不可用时返回预置模板）
- 限流保护（防止 API 滥用）

**4. 合规审核工作流**:
```
AI 生成话术 → 敏感词过滤 → 合规模型审核 → 人工抽检 (10%) → 发布
     ↓            ↓            ↓           ↓
  记录日志    拦截违规    标记风险    审计追踪
```

**审核点**:
- 保险产品名称准确性（AI 模型 + 关键词匹配，100% 审核）
- 收益率/数字合规（正则+AI 审核，100%）
- 绝对化用语（最、第一）（关键词过滤，100%）
- 虚假承诺（保本、稳赚）（AI 语义分析，100%）

**5. 审计日志**:
```json
{
  "audit_log": {
    "user_id": "xxx",
    "action": "generate_copywriting",
    "input": {...},
    "output": {...},
    "review_status": "approved|rejected|pending",
    "timestamp": "2026-04-16T10:00:00Z",
    "ip_address": "x.x.x.x"
  }
}
```

## 五、UI/UX设计

### Q5: UI 状态是如何设计的？

**参考答案**:

采用 5 状态矩阵设计，覆盖所有核心模块：

**内容生成模块**:
| 状态 | 触发条件 | UI 表现 | 用户操作 |
|------|---------|--------|---------|
| Loading | 点击"生成文案"后 | 骨架屏 + 进度条（预计 5-10 秒） | 可取消 |
| Empty | 首次进入/无历史记录 | 引导文案 + 示例展示 | 点击"立即体验" |
| Success | AI 生成完成 | 展示 3 条文案卡片，支持预览/编辑/复制 | 编辑、复制、点赞、重新生成 |
| Error | AI 调用失败/超时 | 错误提示 + 重试按钮 | 重试、切换模板模式 |
| Partial | 部分技能失败 | 展示可用结果 + 失败提示 | 重试失败部分 |

**客户画像模块**: 同样 5 状态设计
**跟进管理模块**: 同样 5 状态设计

## 七、风险管理

### Q7: 项目面临哪些风险？

**参考答案**:

**技术风险**:
- uni-app x 成熟度不足 → MVP 用传统 uni-app，V2 再评估
- Hermes Agent 技能迁移性 → 技能依赖明确模型版本
- 本地模型效果不如云端 → 提供云端/本地切换选项
- 小程序审核不通过 → 提前预审，H5 兜底方案

**合规风险**:
- 保险销售话术合规 → 预置话术经法律顾问审核
- 客户数据隐私保护 → 数据加密存储、权限分级、审计日志
- AI 生成内容责任 → 明确告知 AI 生成，人工确认后再发送

**商业风险**:
- 大厂进场（12-18 个月窗口） → 快速占领细分赛道，建立品牌认知
- 获客成本超预期 → 聚焦行业社群、线下会议等低成本渠道
- 付费转化率低 → 免费试用 + 标杆案例 + 培训转化组合拳

## 八、与 OpenClaw 的对比

### Q8: 为什么选择 Hermes Agent 而不是 OpenClaw？

**参考答案**:

经过深度调研后选择 Hermes Agent，主要原因：

**1. 安全性**:
- OpenClaw: ClawJacked 安全漏洞 (CVE-2026-25253)
- Hermes Agent: 5 层安全防御，无已知高危漏洞

**2. 部署复杂度**:
- OpenClaw: 需要多集群环境（本地笔记本无法运行）
- Hermes Agent: 单节点部署，Docker Compose 一键启动

**3. AI 能力**:
- OpenClaw: LangGraph 状态机，无自进化能力
- Hermes Agent: 自进化技能系统，技能复用降低 70% token 消耗

**4. 面试项目约束**:
- 本地笔记本无多集群环境
- 需要快速产出可演示的 MVP
- Hermes Agent 更符合约束条件
